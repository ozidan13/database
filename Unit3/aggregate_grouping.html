📊 دوال الـ Aggregation الخمسة

The 5 Core Aggregate Functions — Every Edge Case

الدوال دي بتاخد مجموعة من القيم وترجع قيمة واحدة. لكن في تفاصيل مهمة لو مش عارفها هتعدّيك:

COUNT()

عدّ الصفوف

COUNT(*): يعدّ كل الصفوف بما فيها الـ NULLs.
COUNT(col): يعدّ فقط الصفوف اللي قيمتها NOT NULL.
COUNT(DISTINCT col): يعدّ القيم الفريدة.

COUNT(*) → 5 COUNT(email) → 3 -- 2 NULLs تُتجاهل COUNT(DISTINCT dept) → 2
SUM()

مجموع القيم

بيتجاهل الـ NULLs تلقائياً. لو كل القيم NULL → يرجع NULL مش صفر. يشتغل على أرقام فقط.

SUM(salary) → 45000 SUM(bonus) → NULL -- لو كلها NULLs COALESCE(SUM(bonus), 0) → 0 ✅
AVG()

المتوسط الحسابي

بيتجاهل الـ NULLs عند الحساب. ده بيفرق عن SUM/COUNT لأن الـ NULL مش بتُحسب في المقام ولا في البسط.

-- بيانات: 10, 20, NULL, 30 AVG(col) → (10+20+30)/3 = 20 -- مش 60/4 = 15!
MIN() / MAX()

الأصغر والأكبر

بيشتغلوا على أرقام، نصوص، وتواريخ. بيتجاهلوا الـ NULLs. مع النصوص بيستخدموا الترتيب المعجمي (Alphabetical).

MIN(hire_date) → أقدم موظف MAX(name) → آخر اسم أبجدياً MIN(salary) → أقل راتب
STRING_AGG()

دمج النصوص

PostgreSQL-specific — بيدمج قيم عمود في string واحدة مع فاصل. مفيد جداً في التقارير.

STRING_AGG(name, ', ') → 'أحمد, فاطمة, محمد' STRING_AGG(name, ' | ' ORDER BY name)

How NULLs Behave — The Most Important Edge Case

الـ NULL في الـ Aggregates سبب أكبر عدد من الأخطاء في التقارير. لازم تفهم بالظبط بيحصل إيه:

NULL Behavior in Aggregates
-- على البيانات أعلاه: SELECT COUNT(*) AS result, -- 5 (كل الصفوف) COUNT(bonus) AS result, -- 3 (تجاهل الـ NULLs) SUM(bonus) AS result, -- 6500 (2000+3000+1500) AVG(bonus) AS result, -- 6500/3 = 2166.67 ← مش /5! AVG(COALESCE(bonus,0)) AS result; -- 6500/5 = 1300 ← لو عايز المتوسط على الكل -- متوسطين مختلفين لنفس البيانات! -- AVG(bonus) يسألك: "متوسط المكافأة عند من يأخذها؟" → 2166.67 -- AVG(COALESCE(bonus,0)) يسألك: "متوسط المكافأة على كل الموظفين؟" → 1300
💡 كيف تختار؟

AVG(col): "ما المتوسط بين من لديهم قيمة؟" — يتجاهل الـ NULLs
AVG(COALESCE(col, 0)): "ما المتوسط الفعلي على الجميع؟" — يعامل NULL كصفر

مثال عملي: "ما متوسط نقاط التقييم للمنتجات؟"
→ AVG(rating): متوسط المنتجات المُقيَّمة فعلاً
→ AVG(COALESCE(rating, 0)): متوسط شامل المنتجات غير المُقيَّمة (0)

📦 GROUP BY — التجميع

GROUP BY — Single & Multi-Column Grouping

الـ GROUP BY بيقسّم الصفوف لمجموعات حسب قيمة عمود (أو أكتر)، ثم بيطبّق الـ Aggregate على كل مجموعة.

⚠️ قاعدة أساسية في GROUP BY

أي عمود في الـ SELECT لازم يكون:
◆ جزء من GROUP BY، أو
◆ داخل Aggregate Function (COUNT, SUM, إلخ)

❌ SELECT dept_id, name, COUNT(*) FROM employee GROUP BY dept_id → ERROR
✅ إما أضف name لـ GROUP BY، أو استخدمها في STRING_AGG

GROUP BY — Single to Multi-Column
-- ══ GROUP BY عمود واحد ══ SELECT dept_id, COUNT(*) AS headcount, ROUND(AVG(salary), 2) AS avg_salary, SUM(salary) AS total_payroll, MIN(salary) AS min_sal, MAX(salary) AS max_sal FROM employee GROUP BY dept_id ORDER BY total_payroll DESC; -- ══ GROUP BY أعمدة متعددة ══ -- كم موظف في كل قسم وكل مسمى وظيفي؟ SELECT dept_id, job_title, COUNT(*) AS count, AVG(salary) AS avg_sal FROM employee GROUP BY dept_id, job_title -- تركيبة القسم + المسمى ORDER BY dept_id, avg_sal DESC; -- ══ GROUP BY مع Expression ══ -- توزيع الموظفين حسب سنة التوظيف SELECT EXTRACT(YEAR FROM hire_date) AS hire_year, COUNT(*) AS hired_count FROM employee GROUP BY EXTRACT(YEAR FROM hire_date) ORDER BY hire_year; -- ══ GROUP BY مع JOIN ══ SELECT d.dept_name, COUNT(e.emp_id) AS headcount, STRING_AGG(e.full_name, ', ' ORDER BY e.full_name) AS employee_list FROM department d LEFT JOIN employee e ON d.dept_id = e.dept_id GROUP BY d.dept_id, d.dept_name ORDER BY headcount DESC;
🎯 HAVING — فلترة المجموعات

HAVING vs WHERE — The Execution Order Difference

الـ WHERE بيفلتر الصفوف الفردية قبل التجميع. الـ HAVING بيفلتر المجموعات بعد التجميع. مش ممكن تستخدم WHERE على Aggregate Function.

1
FROM + JOINs
→ حدّد الجداول
2
WHERE
→ فلتر الصفوف الفردية هنا (مش ينفع Aggregate)
3
GROUP BY
→ تجميع الصفوف في مجموعات
4
Aggregate Functions
→ COUNT/SUM/AVG/MIN/MAX تُطبَّق على كل مجموعة
5
HAVING
→ فلتر المجموعات بعد التجميع (هنا ينفع Aggregate)
6
SELECT
→ اختيار الأعمدة والتعبيرات
7
ORDER BY
→ ترتيب النتيجة
HAVING vs WHERE — Practical Examples
-- ❌ غلط: WHERE مع Aggregate function SELECT dept_id, COUNT(*) FROM employee WHERE COUNT(*) > 5; -- ERROR: aggregate functions are not allowed in WHERE -- ✅ الصح: HAVING بعد GROUP BY SELECT dept_id, COUNT(*) AS headcount FROM employee GROUP BY dept_id HAVING COUNT(*) > 5; -- الأقسام اللي فيها أكتر من 5 موظفين -- ══ دمج WHERE + GROUP BY + HAVING ══ -- موظفين full-time فقط، مجمّعين بالقسم، إلا الأقسام صغيرة SELECT d.dept_name, COUNT(*) AS headcount, AVG(e.salary) AS avg_salary FROM employee e JOIN department d USING (dept_id) WHERE e.emp_type = 'full_time' -- WHERE قبل التجميع AND e.is_active = TRUE GROUP BY d.dept_id, d.dept_name HAVING COUNT(*) >= 3 -- HAVING بعد التجميع AND AVG(e.salary) > 10000 ORDER BY avg_salary DESC; -- ══ HAVING مع Subquery ══ -- الأقسام اللي متوسط راتبها أعلى من المتوسط العام SELECT dept_id, AVG(salary) AS dept_avg FROM employee GROUP BY dept_id HAVING AVG(salary) > ( SELECT AVG(salary) FROM employee );

🔬 FILTER — الـ Aggregate المشروط

FILTER Clause — Conditional Aggregation

الـ FILTER بيخلّيك تحسب Aggregate على جزء من الصفوف بدون ما تعمل GROUP BY إضافي. مفيد جداً في التقارير المتقاطعة (Pivot-style).

FILTER — Conditional Aggregation vs CASE
-- ══ بدون FILTER — بيانات الأوردرات حسب الشهر والحالة ══ -- الطريقة القديمة: CASE داخل SUM SELECT DATE_TRUNC('month', order_date) AS month, COUNT(*) AS total_orders, SUM(CASE WHEN status = 'delivered' THEN 1 ELSE 0 END) AS delivered, SUM(CASE WHEN status = 'cancelled' THEN 1 ELSE 0 END) AS cancelled FROM orders GROUP BY month; -- ══ نفس النتيجة بـ FILTER — أنظف وأوضح ══ SELECT DATE_TRUNC('month', order_date) AS month, COUNT(*) AS total_orders, COUNT(*) FILTER (WHERE status = 'delivered') AS delivered, COUNT(*) FILTER (WHERE status = 'cancelled') AS cancelled, COUNT(*) FILTER (WHERE status = 'pending') AS pending, SUM(total_amount) FILTER (WHERE status = 'delivered') AS delivered_revenue FROM orders GROUP BY month ORDER BY month; -- ══ FILTER مع AVG ══ SELECT AVG(salary) AS overall_avg, AVG(salary) FILTER (WHERE dept_id = 1) AS dept1_avg, AVG(salary) FILTER (WHERE hire_date >= '2023-01-01') AS new_hires_avg FROM employee;
🧮 ROLLUP و CUBE — التجميع المتدرّج

ROLLUP and CUBE — Subtotals and Cross-Tabulation

الـ ROLLUP بيضيف صفوف Subtotal تلقائياً. الـ CUBE بيعمل كل تركيبات ممكنة. مفيدان في تقارير المبيعات والمالية.

ROLLUP and CUBE
-- ══ ROLLUP — مجاميع فرعية هرمية ══ SELECT COALESCE(dept_name, 'الإجمالي') AS dept, COALESCE(job_title, 'الكل') AS job, COUNT(*) AS count, SUM(salary) AS total FROM employee e JOIN department d USING (dept_id) GROUP BY ROLLUP (dept_name, job_title) ORDER BY dept_name NULLS LAST, job_title NULLS LAST; -- النتيجة: -- IT | Developer | 5 | 50000 -- IT | DBA | 3 | 35000 -- IT | NULL | 8 | 85000 ← Subtotal للـ IT -- HR | Manager | 2 | 30000 -- HR | NULL | 2 | 30000 ← Subtotal للـ HR -- NULL | NULL | 10 | 115000 ← Grand Total -- ══ CUBE — كل التركيبات ══ SELECT dept_id, job_title, hire_year, COUNT(*) AS cnt FROM ( SELECT *, EXTRACT(YEAR FROM hire_date) hire_year FROM employee ) e GROUP BY CUBE (dept_id, job_title, hire_year); -- يعمل كل التركيبات الممكنة: 2³ = 8 مستويات تجميع -- ══ GROUPING() — لتحديد هل العمود هو Rollup NULL أم NULL حقيقية ══ SELECT CASE WHEN GROUPING(dept_id) = 1 THEN 'الإجمالي' ELSE dept_id::TEXT END AS dept, COUNT(*) AS cnt FROM employee GROUP BY ROLLUP (dept_id);

HR Analytics — Complete Report Queries
-- ══════════════════════════════════════ Q1: تقرير الرواتب حسب القسم والمستوى ══════════════════════════════════════ SELECT d.dept_name, e.job_level, COUNT(*) AS headcount, ROUND(AVG(e.salary), 2) AS avg_salary, MIN(e.salary) AS min_salary, MAX(e.salary) AS max_salary, SUM(e.salary) AS total_cost, ROUND(SUM(e.salary) * 100.0 / SUM(SUM(e.salary)) OVER (), 2) AS pct_of_total FROM employee e JOIN department d USING (dept_id) WHERE e.is_active = TRUE GROUP BY d.dept_name, e.job_level HAVING COUNT(*) >= 2 ORDER BY total_cost DESC; -- ══════════════════════════════════════ Q2: معدل الاستقالة الشهرية ══════════════════════════════════════ SELECT TO_CHAR(resignation_date, 'YYYY-MM') AS month, COUNT(*) AS resignations, STRING_AGG(full_name, ', ') AS resigned_employees FROM employee WHERE resignation_date IS NOT NULL AND resignation_date >= DATE_TRUNC('year', CURRENT_DATE) GROUP BY TO_CHAR(resignation_date, 'YYYY-MM') HAVING COUNT(*) > 2 ORDER BY month; -- ══════════════════════════════════════ Q3: توزيع سلم الرواتب (Salary Brackets) ══════════════════════════════════════ SELECT CASE WHEN salary < 5000 THEN 'أقل من 5K' WHEN salary < 10000 THEN '5K - 10K' WHEN salary < 20000 THEN '10K - 20K' ELSE 'أكثر من 20K' END AS salary_bracket, COUNT(*) AS headcount, ROUND(COUNT(*) * 100.0 / SUM(COUNT(*)) OVER (), 1) AS percentage FROM employee WHERE is_active = TRUE GROUP BY salary_bracket ORDER BY MIN(salary);

الجداول المتاحة

orders(order_id, customer_id, order_date, status, total_amount)

category(category_id, category_name)

  • Aاكتب query تجيب إجمالي المبيعات وعدد الأوردرات لكل شهر في 2024 — مع استثناء الأوردرات الملغاة.
  • Bاجيب أعلى 5 فئات (categories) في المبيعات — شرط إن إجمالي مبيعاتها أكثر من 50,000.
  • Cاستخدم FILTER لتجيب في سطر واحد: عدد الأوردرات الـ delivered، الـ pending، والـ cancelled لكل شهر.
E-Commerce Analytics — Solutions
-- A: مبيعات شهرية 2024 (بدون ملغاة) SELECT TO_CHAR(order_date, 'YYYY-MM') AS month, COUNT(*) AS order_count, SUM(total_amount) AS revenue, ROUND(AVG(total_amount), 2) AS avg_order_value FROM orders WHERE status != 'cancelled' AND order_date BETWEEN '2024-01-01' AND '2024-12-31' GROUP BY TO_CHAR(order_date, 'YYYY-MM') ORDER BY month; -- B: أعلى 5 فئات بحد أدنى 50K SELECT c.category_name, SUM(oi.quantity * oi.unit_price) AS total_revenue, COUNT(DISTINCT oi.order_id) AS order_count FROM order_item oi JOIN product p USING (product_id) JOIN category c USING (category_id) GROUP BY c.category_id, c.category_name HAVING SUM(oi.quantity * oi.unit_price) > 50000 ORDER BY total_revenue DESC LIMIT 5; -- C: FILTER للحالات المختلفة في سطر واحد SELECT TO_CHAR(order_date, 'YYYY-MM') AS month, COUNT(*) AS total, COUNT(*) FILTER (WHERE status = 'delivered') AS delivered, COUNT(*) FILTER (WHERE status = 'pending') AS pending, COUNT(*) FILTER (WHERE status = 'cancelled') AS cancelled, ROUND( COUNT(*) FILTER (WHERE status = 'cancelled') * 100.0 / NULLIF(COUNT(*), 0), 1 ) AS cancellation_rate FROM orders GROUP BY TO_CHAR(order_date, 'YYYY-MM') ORDER BY month;
