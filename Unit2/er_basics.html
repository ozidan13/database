أشكال الـ ER Diagram

The Building Blocks

STUDENT

Rectangle — مستطيل

يمثل الـ Entity (الكيان)
كل مستطيل = نوع بيانات مستقل. مثلاً: Student، Course، Department. الكيان هو "حاجة" في الواقع عايزين نخزن بيانات عنها.
Name

Ellipse — بيضاوي

يمثل الـ Attribute (الصفة)
كل بيضاوي = صفة للكيان. مثلاً: اسم الطالب، رقم الهاتف، المعدل. الصفات بتوصف الكيان بالتفصيل.
enrolls

Diamond — معين

يمثل الـ Relationship (العلاقة)
كل معين = علاقة بين كيانين أو أكتر. مثلاً: الطالب enrolls in مادة. الدكتور teaches مادة.

الكيانات بالتفصيل

Entity Types — Strong vs Weak

الـ Entity (الكيان) هو أي حاجة في العالم الحقيقي عايزين نخزن بيانات عنها.

في نوعين:

📦 Strong Entity — كيان قوي

بيقدر يعيش لوحده — عنده Primary Key خاص بيه.

أمثلة: Student (ID)، Department (Dept_ID)، Course (Course_Code)

الرسم: مستطيل عادي بخط واحد 📦

👶 Weak Entity — كيان ضعيف

مش بيقدر يعيش لوحده — محتاج كيان تاني عشان يتعرف. عنده Partial Key بس.

مثال: Dependent (معال الموظف) — "أحمد ابن محمد" مش هنعرفه إلا لو عرفنا مين "محمد" (الموظف).

الرسم: مستطيل بخط مزدوج 📦📦

🎓 مهم للامتحان: الـ Weak Entity دايماً بيكون في علاقة Total Participation مع الـ Owner Entity. والعلاقة بينهم اسمها Identifying Relationship (بترسم بمعين مزدوج).

أنواع الصفات

Attribute Types

🔵

Simple Attribute

صفة بسيطة

صفة مش ممكن نقسمها لأجزاء أصغر. مثلاً: Age، GPA، SSN.

🟣

Composite Attribute

صفة مركبة

صفة ممكن نقسمها. مثلاً: Full Name = First + Middle + Last. أو Address = شارع + مدينة + رقم.

🔴

Multivalued Attribute

صفة متعددة القيم

ممكن يكون لها أكتر من قيمة. مثلاً: Phone Numbers (ممكن يكون عند الشخص 2-3 أرقام). بترسم ببيضاوي مزدوج.

🟡

Derived Attribute

صفة مشتقة

قيمتها بتتحسب من صفات تانية. مثلاً: Age ممكن نحسبه من Birth Date. بترسم ببيضاوي متقطع.

🔑

Key Attribute

صفة مفتاحية

الصفة اللي بتميز كل Entity عن التاني — زي Student ID أو SSN. بترسم ببيضاوي مع خط تحت الاسم.

📦

Stored Attribute

صفة مخزنة

الصفة اللي فعلاً متخزنة في الـ Database. مثلاً: Birth Date مخزن — بس Age مشتق منه.

🎨 مثال عملي: كيان الطالب

الخلاصة

Key Takeaways

① الـ ER Diagram هو الخريطة اللي بترسمها قبل ما تبني الـ Database.

② الأشكال الأساسية: مستطيل (كيان) + بيضاوي (صفة) + معين (علاقة).

③ الصفات ليها أنواع: بسيطة، مركبة، متعددة القيم، مشتقة، مفتاحية.

④ في الدرس الجاي هندرس العلاقات بالتفصيل — Cardinality و Participation!
