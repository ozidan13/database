🔌 Connection Pool — بركة الاتصالات

Why Pooling? — Managing Database Connections Efficiently

فتح connection جديد لكل request = بطيء جداً (50-200ms لكل اتصال). الـ Pool بيعمل مجموعة من الاتصالات الجاهزة ويُعيد استخدامها — بينقص وقت الاستجابة بشكل جذري.

src/db/pool.ts — Connection Pool Setup
import { Pool, PoolClient } from 'pg'; import 'dotenv/config'; const pool = new Pool({ host: process.env.DB_HOST || 'localhost', port: Number(process.env.DB_PORT) || 5432, database: process.env.DB_NAME || 'ecommerce_db', user: process.env.DB_USER || 'postgres', password: process.env.DB_PASSWORD || '', // Pool configuration max: Number(process.env.DB_MAX_CONNECTIONS) || 20, min: 2, // الحد الأدنى من الاتصالات الجاهزة دائماً idleTimeoutMillis: 30000, // اتصال خامل يُغلق بعد 30 ثانية connectionTimeoutMillis: 5000,// ينتظر 5 ثوانٍ إذا الـ Pool ممتلئة allowExitOnIdle: true, }); // مراقبة أحداث الـ Pool pool.on('connect', (client: PoolClient) => { console.log('New DB connection established'); }); pool.on('error', (err: Error) => { console.error('Unexpected DB pool error:', err); process.exit(-1); // اخرج — الـ Pool في حالة خطأ }); // Helper: تنفيذ query بسيطة export const query = (text: string, params?: any[]) => pool.query(text, params); // Helper: فتح client للـ Transactions export const getClient = () => pool.connect(); export default pool;

CRUD, Parameterized Queries, Transactions in Node.js

src/repositories/customer.repo.ts — Full CRUD
import { query, getClient } from '../db/pool'; export interface Customer { customer_id: number; email: string; full_name: string; phone?: string; } export class CustomerRepository { // ══ READ ALL with pagination ══ async findAll(page = 1, limit = 20): Promise<Customer[]> { const offset = (page - 1) * limit; const result = await query( `SELECT customer_id, email, full_name, phone FROM customer WHERE is_active = true ORDER BY created_at DESC LIMIT $1 OFFSET $2`, [limit, offset] ); return result.rows; } // ══ READ ONE by ID ══ async findById(id: number): Promise<Customer | null> { const result = await query( `SELECT * FROM customer WHERE customer_id = $1 AND is_active = true`, [id] ); return result.rows[0] || null; } // ══ CREATE ══ async create(data: Pick<Customer, 'email'|'full_name'|'phone'> & { password_hash: string }): Promise<Customer> { const result = await query( `INSERT INTO customer (email, full_name, phone, password_hash) VALUES ($1, $2, $3, $4) RETURNING customer_id, email, full_name, phone`, [data.email, data.full_name, data.phone, data.password_hash] ); return result.rows[0]; } // ══ UPDATE ══ async update(id: number, data: Partial<Pick<Customer, 'full_name'|'phone'>>): Promise<Customer | null> { const fields: string[] = []; const values: any[] = []; let idx = 1; if (data.full_name) { fields.push(`full_name = $` + idx++); values.push(data.full_name); } if (data.phone) { fields.push(`phone = $` + idx++); values.push(data.phone); } if (!fields.length) return null; values.push(id); const result = await query( `UPDATE customer SET ${fields.join(', ')}, updated_at=NOW() WHERE customer_id = ${idx} AND is_active=true RETURNING *`, values ); return result.rows[0] || null; } // ══ SOFT DELETE ══ async softDelete(id: number): Promise<boolean> { const result = await query( `UPDATE customer SET is_active=false, updated_at=NOW() WHERE customer_id=$1`, [id] ); return result.rowCount! > 0; } // ══ TRANSACTION: تسجيل عميل + إنشاء عنوان في نفس الوقت ══ async registerWithAddress(customerData: any, addressData: any): Promise<Customer> { const client = await getClient(); try { await client.query('BEGIN'); const custResult = await client.query( `INSERT INTO customer (email, full_name, password_hash) VALUES ($1, $2, $3) RETURNING *`, [customerData.email, customerData.full_name, customerData.password_hash] ); const customer = custResult.rows[0]; await client.query( `INSERT INTO address (customer_id, line1, city, is_default) VALUES ($1, $2, $3, true)`, [customer.customer_id, addressData.line1, addressData.city] ); await client.query('COMMIT'); return customer; } catch (err) { await client.query('ROLLBACK'); throw err; } finally { client.release(); // دايماً أعد الـ client للـ pool! } } }
🛡️ SQL Injection — الخطر الأكبر

Parameterized Queries — The Only Real Defense Against SQL Injection

SQL Injection — The Attack and Defense
// ❌ VULNERABLE — Don't ever do this! const userId = req.params.id; // يمكن أن يكون: "1 OR 1=1" const result = await query( `SELECT * FROM customer WHERE customer_id = ` + userId // → SELECT * FROM customer WHERE customer_id = 1 OR 1=1 // → يعطي كل السجلات! كارثة! ); // ❌ VULNERABLE — string interpolation const name = req.body.name; // يمكن أن يكون: "'; DROP TABLE customer; --" await query(`SELECT * FROM customer WHERE full_name = '${name}'`); // ✅ SAFE — Parameterized Query دائماً! const result = await query( `SELECT * FROM customer WHERE customer_id = $1`, [req.params.id] // pg يُعالجه كـ literal value — ليس كـ SQL code ); // ✅ SAFE — دايماً استخدم $1, $2... لكل قيمة خارجية await query( `UPDATE customer SET full_name=$1, phone=$2 WHERE customer_id=$3`, [req.body.name, req.body.phone, req.params.id] );
⚠️ القاعدة الذهبية: لا تُدمج أبداً قيمة المستخدم مع SQL كـ string

كل قيمة تأتي من المستخدم (params, body, query string, headers) يجب أن تمرّ عبر Parameterized Query — بدون استثناء. الـ Parameterized Queries مضمونة على مستوى Driver — ليس مجرد escape.

🏗️ Repository Pattern + Express Routes

Repository Pattern — Separating Data Access from Business Logic

src/routes/customer.routes.ts — Clean REST API
import { Router, Request, Response, NextFunction } from 'express'; import { CustomerRepository } from '../repositories/customer.repo'; const router = Router(); const customerRepo = new CustomerRepository(); // GET /api/customers?page=1&limit=20 router.get('/', async (req: Request, res: Response, next: NextFunction) => { try { const page = Number(req.query.page) || 1; const limit = Number(req.query.limit) || 20; const customers = await customerRepo.findAll(page, limit); res.json({ data: customers, page, limit }); } catch (err) { next(err); } }); // GET /api/customers/:id router.get('/:id', async (req, res, next) => { try { const customer = await customerRepo.findById(Number(req.params.id)); if (!customer) return res.status(404).json({ error: 'Customer not found' }); res.json({ data: customer }); } catch (err) { next(err); } }); // POST /api/customers router.post('/', async (req, res, next) => { try { const customer = await customerRepo.create(req.body); res.status(201).json({ data: customer }); } catch (err: any) { if (err.code === '23505') // unique_violation return res.status(409).json({ error: 'Email already exists' }); next(err); } }); // DELETE /api/customers/:id (soft delete) router.delete('/:id', async (req, res, next) => { try { const deleted = await customerRepo.softDelete(Number(req.params.id)); if (!deleted) return res.status(404).json({ error: 'Not found' }); res.status(204).send(); } catch (err) { next(err); } }); // Global Error Handler Middleware export const errorHandler = (err: any, req: Request, res: Response, next: NextFunction) => { console.error('DB Error:', err); const pgErrors: Record<string, number> = { '23502': 400, // not_null_violation '23503': 409, // foreign_key_violation '23505': 409, // unique_violation '23514': 400, // check_violation }; const status = pgErrors[err.code] || 500; res.status(status).json({ error: err.message || 'Internal Server Error' }); }; export default router;

Solution — ProductRepository + Order Transaction
// A: ProductRepository export class ProductRepository { async findAll(categoryId?: number, page=1, limit=20) { const params: any[] = [limit, (page-1)*limit]; let sql = `SELECT p.*, c.name AS category_name FROM product p JOIN category c USING(category_id) WHERE p.is_active = true`; if (categoryId) { sql += ` AND p.category_id = $3`; params.push(categoryId); } sql += ` ORDER BY p.created_at DESC LIMIT $1 OFFSET $2`; return (await query(sql, params)).rows; } async updatePrice(id: number, price: number) { if (price < 0) throw new Error('Price must be non-negative'); const r = await query( `UPDATE product SET price=$1 WHERE product_id=$2 AND is_active=true RETURNING *`, [price, id] ); return r.rows[0] || null; } } // B: POST /api/orders — Transaction router.post('/orders', async (req, res, next) => { const client = await getClient(); try { await client.query('BEGIN'); const { customer_id, address_id, items } = req.body; const total = items.reduce((s: number, i: any) => s + i.quantity * i.unit_price, 0); const orderRes = await client.query( `INSERT INTO orders(customer_id,address_id,total_amount) VALUES($1,$2,$3) RETURNING *`, [customer_id, address_id, total] ); const order = orderRes.rows[0]; for (const item of items) { await client.query( `INSERT INTO order_item(order_id,product_id,quantity,unit_price) VALUES($1,$2,$3,$4)`, [order.order_id, item.product_id, item.quantity, item.unit_price] ); } await client.query('COMMIT'); res.status(201).json({ data: order }); } catch(err) { await client.query('ROLLBACK'); next(err); } finally { client.release(); } });
🧠 اختبار الفهم
١. لماذا نستخدم Connection Pool بدل فتح connection جديد لكل request؟
لأن الـ Pool أسهل في الكتابة
فتح connection جديد يستغرق 50-200ms. الـ Pool يُعيد استخدام الاتصالات الجاهزة — يقلل وقت الاستجابة بشكل جذري ويمنع استنزاف resources.
الـ Pool أكثر أماناً من SQL Injection
٢. ما الفرق بين pool.query() وpool.connect()؟
لا فرق
pool.query(): ينفذ query واحدة ويُعيد الـ connection تلقائياً. pool.connect(): يُعطيك client لعدة queries (Transactions) — يجب release() يدوياً.
pool.connect() أسرع دائماً
٣. ما PostgreSQL error code لـ Unique Violation؟
23502
23505 — unique_violation. 23502=not_null, 23503=foreign_key, 23514=check_violation.
23503
٤. لماذا يجب استدعاء client.release() في finally؟
لحذف البيانات
لإعادة الـ client للـ Pool — بدون release() الـ Pool ينفد من الاتصالات وتتوقف الـ App كلها.
