Prisma ORM

Type-Safe Database Access — Schema-First Modern ORM for Node.js

Prisma هو الـ ORM الأقوى في عالم Node.js/TypeScript. Type-safe بالكامل، auto-completion كامل، وSchema declarative واضح — يجعل التعامل مع قاعدة البيانات أمرا ممتعاً.

Prisma vs Raw pg — Type Safety, DX, and When to Use Each

Raw pg (pg library)

  • SQL كامل — قوة لا محدودة
  • لا type safety — أخطاء runtime
  • تكتب Query Builders يدوياً
  • أسرع في queries المعقدة
  • مناسب للـ DBA والـ Analytics

Prisma ORM

  • Type-safe بالكامل — أخطاء compile-time
  • Auto-completion في IDE
  • Migration system مدمج
  • Prisma Studio — UI لقاعدة البيانات
  • مثالي للـ Backend Teams

🧠 اختبار الفهم
١. ما الفرق بين include وselect في Prisma؟
لا فرق — مترادفان
include: يجلب العلاقات (relations) مع كل حقولها. select: يحدد الحقول المطلوبة فقط (يمكن استخدامه داخل include أيضاً).
select للـ WHERE وinclude للـ ORDER BY
٢. ما الفرق بين findUnique وfindFirst؟
لا فرق
findUnique: يعمل فقط على @unique/@id fields — أسرع (index scan مضمون). findFirst: يعمل على أي where condition — يُرجع أول نتيجة.
findUnique يُرجع array
٣. لماذا نستخدم Singleton Pattern مع PrismaClient؟
لتسريع الـ Queries
لأن كل PrismaClient instance يفتح Connection Pool جديد. في Development مع Hot Reload، سيُنشئ instances متعددة — Singleton يمنع استنزاف الاتصالات.
لأن Prisma لا يدعم multiple instances
٤. متى تستخدم Interactive Transaction بدل Sequential Transaction؟
دائماً — Interactive أفضل في كل الحالات
عند عمليات تعتمد على نتائج بعضها (مثل استخدام id من create في خطوة لاحقة). Sequential للعمليات المستقلة المتزامنة — أسرع.
