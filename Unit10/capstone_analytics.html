Monthly Revenue KPIs مع MoM Growth
SELECT month, gmv, aov, order_count, unique_buyers, LAG(gmv) OVER (ORDER BY month) AS prev_gmv, ROUND((gmv - LAG(gmv) OVER (ORDER BY month)) * 100.0 / NULLIF(LAG(gmv) OVER (ORDER BY month), 0), 2) AS mom_growth_pct, SUM(gmv) OVER (ORDER BY month) AS cumulative_gmv, ROUND(gmv - AVG(gmv) OVER (ORDER BY month ROWS BETWEEN 2 PRECEDING AND CURRENT ROW), 2) AS vs_3mo_avg FROM ( SELECT DATE_TRUNC('month', order_date) AS month, SUM(total_amount) AS gmv, ROUND(AVG(total_amount), 2) AS aov, COUNT(*) AS order_count, COUNT(DISTINCT user_id) AS unique_buyers FROM orders WHERE status = 'delivered' GROUP BY 1 ) monthly ORDER BY month;
👥 Cohort Analysis — تحليل الاحتفاظ بالعملاء

Monthly Cohort Retention — Which Acquisition Month Retains Best?

Cohort Analysis تُجيب على: "من كل 100 عميل اشتروا في يناير، كم ظلّوا يشترون في شهر 2، 3، 6؟" — يُكشف عن جودة التجربة والاستحواذ.

Monthly Retention Cohort
WITH first_order AS ( SELECT user_id, DATE_TRUNC('month', MIN(order_date)) AS cohort_month FROM orders WHERE status = 'delivered' GROUP BY 1 ), monthly_activity AS ( SELECT fo.cohort_month, DATE_TRUNC('month', o.order_date) AS activity_month, COUNT(DISTINCT o.user_id) AS active_users FROM orders o JOIN first_order fo ON fo.user_id = o.user_id WHERE o.status = 'delivered' GROUP BY 1, 2 ), cohort_size AS ( SELECT cohort_month, COUNT(DISTINCT user_id) AS total FROM first_order GROUP BY 1 ) SELECT ma.cohort_month, EXTRACT('month' FROM AGE(ma.activity_month, ma.cohort_month)) AS months_since_join, cs.total AS cohort_size, ma.active_users, ROUND(ma.active_users * 100.0 / cs.total, 1) AS retention_pct FROM monthly_activity ma JOIN cohort_size cs USING (cohort_month) ORDER BY ma.cohort_month, months_since_join;
🔻 Conversion Funnel

Purchase Funnel — View > Cart > Checkout > Payment — Drop-off Analysis

Purchase Funnel — Conversion & Drop-off
-- user_events: {user_id, event, product_id, session_id, created_at} WITH funnel AS ( SELECT COUNT(DISTINCT CASE WHEN event = 'product_view' THEN user_id END) AS views, COUNT(DISTINCT CASE WHEN event = 'add_to_cart' THEN user_id END) AS carts, COUNT(DISTINCT CASE WHEN event = 'checkout_start' THEN user_id END) AS checkouts, COUNT(DISTINCT CASE WHEN event = 'purchase' THEN user_id END) AS purchases FROM user_events WHERE created_at >= NOW() - INTERVAL '30 days' ) SELECT views, carts, checkouts, purchases, ROUND(carts * 100.0 / NULLIF(views, 0), 1) AS view_to_cart_pct, ROUND(checkouts * 100.0 / NULLIF(carts, 0), 1) AS cart_to_checkout_pct, ROUND(purchases * 100.0 / NULLIF(checkouts, 0), 1) AS checkout_to_purchase_pct, ROUND(purchases * 100.0 / NULLIF(views, 0), 2) AS overall_cvr FROM funnel;
🎯 RFM Segmentation — تصنيف العملاء

Recency, Frequency, Monetary — Customer Segmentation for Marketing

RFM هو الأساس في الـ Marketing Analytics — كل عميل يحصل على Score لـ: Recency (آخر شراء)، Frequency (عدد المرات)، Monetary (إجمالي الإنفاق).

RFM Score وCustomer Segments
WITH rfm_raw AS ( SELECT user_id, MAX(order_date) AS last_order_date, COUNT(*) AS frequency, SUM(total_amount) AS monetary, EXTRACT('day' FROM NOW() - MAX(order_date)) AS recency_days FROM orders WHERE status = 'delivered' GROUP BY 1 ), rfm_scored AS ( SELECT user_id, recency_days, frequency, monetary, NTILE(5) OVER (ORDER BY recency_days ASC) AS r_score, -- أقل = أحدث = أفضل NTILE(5) OVER (ORDER BY frequency DESC) AS f_score, NTILE(5) OVER (ORDER BY monetary DESC) AS m_score FROM rfm_raw ) SELECT user_id, recency_days, frequency, ROUND(monetary, 2) AS monetary, r_score, f_score, m_score, CASE WHEN r_score = 5 AND f_score = 5 THEN '🏆 Champions' WHEN r_score >= 4 AND f_score >= 4 THEN '💎 Loyal Customers' WHEN r_score = 5 THEN '🆕 Recent Customers' WHEN r_score >= 3 AND m_score >= 4 THEN '💰 Potential Loyalists' WHEN r_score <= 2 AND f_score >= 3 THEN '⚠️ At Risk' WHEN r_score <= 1 THEN '💤 Lost Customers' ELSE '🔄 Need Attention' END AS segment FROM rfm_scored ORDER BY r_score DESC, f_score DESC;
🧮 OLAP — CUBE & ROLLUP للتقارير المتعددة

GROUPING SETS, ROLLUP, CUBE — Multi-Dimensional Analysis in One Query

ROLLUP — Subtotals per Month, Category, Grand Total
-- ROLLUP: يُولّد Subtotals تلقائياً (Month → Category → Grand Total) SELECT DATE_TRUNC('month', o.order_date) AS month, p.category, COUNT(*) AS orders, SUM(oi.unit_price * oi.quantity) AS revenue, GROUPING(DATE_TRUNC('month', o.order_date)) AS is_month_total, GROUPING(p.category) AS is_category_total FROM orders o JOIN order_items oi ON oi.order_id = o.id JOIN products_search p ON p.mongo_id = oi.product_id WHERE o.status = 'delivered' GROUP BY ROLLUP (DATE_TRUNC('month', o.order_date), p.category) ORDER BY month NULLS LAST, category NULLS LAST; -- CUBE: كل Combination ممكنة (Month×Category، Month فقط، Category فقط، الكل) GROUP BY CUBE (month, category, country) -- 2^3 = 8 combinations -- GROUPING SETS: اختر بالضبط اللي تريده GROUP BY GROUPING SETS ( (month, category), -- Per Month+Category (month), -- Per Month only () -- Grand Total )
