✏️ CRUD Operations

Create, Read, Update, Delete — MongoDB Shell & Node.js

MongoDB CRUD — Shell Commands
// ══ CREATE ══ db.products.insertOne({ name: "iPhone 15", price: 45000, stock: 100 }); db.products.insertMany([ { name: "Samsung S24", price: 38000 }, { name: "Pixel 8", price: 30000 } ]); // ══ READ — Query Operators ══ db.products.find({ price: { $gte: 30000, $lte: 50000 } }); db.products.find({ category: "Electronics", stock: { $gt: 0 } }); db.products.find({ tags: { $in: ["gaming", "laptop"] } }); db.products.find({ "specs.ram": "32GB" }); // Nested field query db.products.find({ $or: [{ price: { $lt: 10000 } }, { stock: 0 }] }); // Projection, Sort, Limit, Skip db.products .find({ category: "Electronics" }, { name: 1, price: 1, _id: 0 }) .sort({ price: -1 }) // -1 = DESC .skip(20) .limit(10); // ══ UPDATE ══ db.products.updateOne( { _id: ObjectId("64f1...") }, { $set: { price: 32000, "specs.ram": "64GB" } } ); db.products.updateMany( { category: "Electronics" }, { $inc: { stock: -1 } } // يُنقص الـ stock بـ 1 ); // Update Operators: $set، $unset، $inc، $push، $pull، $addToSet db.products.findOneAndUpdate( { _id: ObjectId("64f1...") }, { $push: { tags: "sale" } }, { returnDocument: "after" } ); // ══ DELETE ══ db.products.deleteOne({ _id: ObjectId("64f1...") }); db.products.deleteMany({ stock: 0, is_discontinued: true });
🔬 Aggregation Pipeline

MongoDB Aggregation — The Equivalent of SQL GROUP BY + JOINs + Window Functions

الـ Aggregation Pipeline هو أقوى ميزة في MongoDB — تُمرّر Documents عبر مراحل (Stages) للتحويل والتحليل، مثل Unix Pipes.

Aggregation Pipeline — Sales Report by Category
db.orders.aggregate([ // Stage 1: $match — فلتر (مثل WHERE) { $match: { status: "delivered", order_date: { $gte: new Date("2024-01-01") } }}, // Stage 2: $lookup — JOIN مع collection أخرى { $lookup: { from: "products", localField: "product_id", foreignField: "_id", as: "product" }}, { $unwind: "$product" }, // Array → Single Document // Stage 3: $group — GROUP BY { $group: { _id: "$product.category", total_revenue: { $sum: { $multiply: ["$quantity", "$unit_price"] } }, order_count: { $sum: 1 }, avg_order_val: { $avg: "$total" }, unique_customers: { $addToSet: "$customer_id" } }}, // Stage 4: $addFields — حقل محسوب { $addFields: { unique_customer_count: { $size: "$unique_customers" } }}, // Stage 5: $sort وLimit { $sort: { total_revenue: -1 } }, { $limit: 10 }, // Stage 6: $project — اختر الحقول { $project: { category: "$_id", total_revenue: 1, order_count: 1, unique_customer_count: 1, _id: 0 }} ]); // أهم Stages: $match، $group، $lookup، $unwind، $project、$sort، $limit، $facet
🗂️ Indexes في MongoDB

MongoDB Indexes — Single, Compound, Text, Geospatial, TTL

MongoDB Indexes — أهم الأنواع
// Single Field Index db.products.createIndex({ price: 1 }); // 1=ASC, -1=DESC // Compound Index (نفس قواعد ESR Rule في PostgreSQL) db.products.createIndex({ category: 1, price: -1 }); // Unique Index db.users.createIndex({ email: 1 }, { unique: true }); // Text Index للـ Full-Text Search db.products.createIndex({ name: "text", description: "text" }); db.products.find({ $text: { $search: "laptop gaming dell" } }, { score: { $meta: "textScore" } }) .sort({ score: { $meta: "textScore" } }); // TTL Index — حذف تلقائي بعد مدة (للـ Sessions، Logs) db.sessions.createIndex( { created_at: 1 }, { expireAfterSeconds: 3600 } // حذف بعد ساعة ); // Partial Index (نفس فكرة PostgreSQL) db.orders.createIndex( { customer_id: 1, order_date: -1 }, { partialFilterExpression: { status: { $in: ["pending", "shipped"] } } } ); // Geospatial Index للمواقع db.restaurants.createIndex({ location: "2dsphere" }); db.restaurants.find({ location: { $near: { $geometry: { type: "Point", coordinates: [31.2357, 30.0444] }, // Cairo $maxDistance: 5000 // 5km } } });
🏗️ Schema Design — Embedding vs Referencing

The Most Important MongoDB Design Decision — Embed or Reference?

📌 Embedding — متى تستخدمه؟

✅ البيانات تُقرأ معاً دائماً (One Query بدلاً من JOIN)
✅ علاقة 1:1 أو 1:N لعدد محدود صغير
✅ البيانات الفرعية لا معنى لها بدون الـ Parent
مثال: User Profile مع Address وSocial Links
⚠️ حد Document في MongoDB = 16MB

🔗 Referencing — متى تستخدمه؟

✅ البيانات الفرعية كثيرة جداً (Orders لـ User)
✅ البيانات مُستخدمة بشكل مستقل كثيراً
✅ Many-to-Many relationships
مثال: Product → Category (Reference لأن Category تُقرأ بشكل مستقل)
استخدم $lookup في Aggregation للـ JOIN

Mongoose ODM — Node.js Schema + Model
import mongoose from 'mongoose'; // Schema Definition const productSchema = new mongoose.Schema({ name: { type: String, required: true, trim: true }, price: { type: Number, required: true, min: 0 }, category: { type: String, enum: ['electronics', 'clothing', 'food'] }, tags: [String], specs: mongoose.Schema.Types.Mixed, // Any shape seller: { type: mongoose.Schema.Types.ObjectId, ref: 'User' } }, { timestamps: true }); // auto createdAt, updatedAt // Indexes productSchema.index({ price: 1, category: 1 }); productSchema.index({ name: 'text' }); // Virtual Field productSchema.virtual('priceInUSD').get(function() { return (this.price / 48).toFixed(2); }); export const Product = mongoose.model('Product', productSchema); // Usage — Populate (JOIN-like) const products = await Product .find({ category: 'electronics', price: { $gte: 1000 } }) .populate('seller', 'name email') // JOIN مع Users .sort({ price: -1 }) .limit(20) .lean(); // Plain JS Object — أسرع من Mongoose Document
🧠 اختبار الفهم
١. عندك User لديه ملايين Orders. هل تعمل Embedding أم Referencing للـ Orders؟
Embedding — أسرع
Referencing — الـ Embedding يتجاوز حد 16MB بسرعة مع ملايين Orders. Orders تُقرأ وتُبحث بشكل مستقل. الحل: orders collection منفصلة مع user_id reference.
لا فرق
٢. ما الـ TTL Index ومتى تستخدمه؟
Index على حقل الوقت فقط
يحذف Documents تلقائياً بعد مدة محددة (expireAfterSeconds). مثالي للـ Sessions (انتهاء Timeout)، Verification Tokens، Cache Data، Logs مؤقتة — بدون Cron Job يدوي.
Index يُسرّع Query الوقت فقط
٣. في Aggregation Pipeline، ما وظيفة $unwind؟
يُلغي الـ Pipeline
يُحوّل Array field إلى Documents منفصلة — كل عنصر في الـ Array يُصبح Document مستقل. ضروري بعد $lookup لأنه يُرجع Array، وقبل $group على عناصر Array.
يُزيل الـ Duplicate Documents
٤. ما الفرق بين .find() العادي و.lean() في Mongoose؟
lean() يُرجع نتائج أقل
بدون .lean(): يُرجع Mongoose Documents كاملة (مع Methods، Virtuals، Validators) — أثقل. مع .lean(): يُرجع Plain JavaScript Objects — أخف بـ 5-10x وأسرع. استخدمه في القراءة فقط حيث لا تحتاج Mongoose Methods.
lean() أبطأ لأنه يُحوّل البيانات
