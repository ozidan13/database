Window Functions — دوال النافذة

OVER, PARTITION BY, ROW_NUMBER, RANK, LAG, LEAD, Running Totals, Frame Clauses

الـ Window Functions هي أقوى ميزة في SQL. بتديك قوة الـ Aggregation بدون ما تفقد تفاصيل كل صف — كل عمليات الـ Analytics تمر منها.

🤔 لماذا Window Functions؟

The Power of Knowing Context Without Losing Details

تخيّل إنك عايز تعرف راتب كل موظف ونسبته من متوسط قسمه في نفس الوقت. بالـ GROUP BY، أنت تخسر تفاصيل الموظف الفردي. الـ Window Functions بتحسب على "نافذة" من الصفوف بدون إزالة التفاصيل.

Window vs GROUP BY — The Difference
-- ❌ GROUP BY: تفقد تفاصيل الموظف SELECT dept_id, AVG(salary) AS dept_avg FROM employee GROUP BY dept_id; -- بيرجع صف واحد لكل قسم — مش تفاصيل كل موظف -- ✅ Window Function: كل موظف + متوسط قسمه SELECT full_name, salary, dept_id, AVG(salary) OVER (PARTITION BY dept_id) AS dept_avg, salary - AVG(salary) OVER (PARTITION BY dept_id) AS diff FROM employee; -- كل صف موظف محتفظ بتفاصيله + عمود متوسط قسمه
🔬 تشريح الـ OVER Clause

Anatomy of the OVER() Window Specification

FUNCTION_NAME() OVER ( [PARTITION BY ...] [ORDER BY ...] [frame_clause] )

FUNCTION() الدالة: ROW_NUMBER, RANK, SUM, AVG, LAG...
OVER الكلمة المفتاحية — بتحوّل الدالة لـ Window Function
PARTITION BY col تقسيم الصفوف لمجموعات (مثل GROUP BY لكن بدون دمج). اختياري.
ORDER BY col ترتيب الصفوف داخل كل partition — مطلوب لـ Ranking وOffset functions.
frame_clause تحديد "نطاق" الصفوف: ROWS BETWEEN ... AND ... — اختياري.
OVER Clause Combinations
-- OVER() بدون فلوس: النافذة = كل الجدول RANK() OVER () -- OVER مع PARTITION فقط AVG(salary) OVER (PARTITION BY dept_id) -- OVER مع ORDER BY فقط (بدون partition) ROW_NUMBER() OVER (ORDER BY hire_date) -- OVER مع الاثنين RANK() OVER (PARTITION BY dept_id ORDER BY salary DESC) -- WINDOW ALIAS — تعريف نافذة مرة وإعادة استخدامها SELECT full_name, ROW_NUMBER() OVER dept_win, RANK() OVER dept_win, AVG(salary) OVER dept_win FROM employee WINDOW dept_win AS (PARTITION BY dept_id ORDER BY salary DESC);
🏆 Ranking Functions

ROW_NUMBER, RANK, DENSE_RANK, NTILE — When to Use Each

ROW_NUMBER()

رقم تسلسلي فريد

بيعطي كل صف رقماً تسلسلياً مستمراً. حتى لو في تعادل → أرقام مختلفة. الأكثر استخداماً لـ Pagination وإزالة التكرار.

ROW_NUMBER() OVER (ORDER BY salary DESC) → 1, 2, 3, 4, 5 (حتى لو في تعادل)
RANK()

الترتيب مع فجوات

في حالة التعادل: نفس الرقم. الرقم التالي يقفز. مثل سباق: اتنين جاوا أول → مفيش ثاني، التالت هو الثالث.

RANK() OVER (ORDER BY salary DESC) → 1, 1, 3, 4 (الثاني غائب)
DENSE_RANK()

الترتيب بدون فجوات

في حالة التعادل: نفس الرقم. لكن الرقم التالي مستمر. مثالي لـ "أعلى N" بدون فجوات في الترتيب.

DENSE_RANK() OVER (ORDER BY salary DESC) → 1, 1, 2, 3 (بدون فجوات)
NTILE(n)

تقسيم لـ n مجموعات

بيقسّم الصفوف لـ n مجموعات متساوية (Quartiles, Deciles). مفيد للتحليل الإحصائي والـ Segmentation.

NTILE(4) OVER (ORDER BY salary) → الربع الأول/الثاني/الثالث/الرابع

Ranking Functions — Practical Use Cases
-- ══ أعلى 3 موظفين في كل قسم ══ SELECT dept_id, full_name, salary, dept_rank FROM ( SELECT dept_id, full_name, salary, DENSE_RANK() OVER (PARTITION BY dept_id ORDER BY salary DESC) AS dept_rank FROM employee WHERE is_active = TRUE ) AS ranked WHERE dept_rank <= 3 ORDER BY dept_id, dept_rank; -- ══ Deduplication — أحدث أوردر لكل عميل ══ SELECT customer_id, order_id, order_date, total_amount FROM ( SELECT *, ROW_NUMBER() OVER ( PARTITION BY customer_id ORDER BY order_date DESC ) AS rn FROM orders ) ranked WHERE rn = 1; -- أحدث أوردر فقط لكل عميل -- ══ Quartiles — توزيع الرواتب ══ SELECT full_name, salary, NTILE(4) OVER (ORDER BY salary) AS quartile, CASE NTILE(4) OVER (ORDER BY salary) WHEN 1 THEN 'Q1 - أدنى 25%' WHEN 2 THEN 'Q2 - 25-50%' WHEN 3 THEN 'Q3 - 50-75%' WHEN 4 THEN 'Q4 - أعلى 25%' END AS quartile_label FROM employee;
📌 FIRST_VALUE, LAST_VALUE, NTH_VALUE

Value Window Functions — Accessing Specific Rows in Window

FIRST_VALUE / LAST_VALUE / NTH_VALUE
-- FIRST_VALUE: أعلى راتب في القسم (على كل صف) SELECT full_name, salary, dept_id, FIRST_VALUE(salary) OVER ( PARTITION BY dept_id ORDER BY salary DESC ) AS dept_highest_salary, FIRST_VALUE(full_name) OVER ( PARTITION BY dept_id ORDER BY salary DESC ) AS top_earner_name FROM employee; -- LAST_VALUE: ⚠️ تحتاج RANGE BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING -- بدونها بترجع القيمة الحالية مش الأخيرة! SELECT full_name, salary, LAST_VALUE(salary) OVER ( PARTITION BY dept_id ORDER BY salary DESC ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING ) AS dept_lowest_salary FROM employee; -- NTH_VALUE: الراتب الثاني الأعلى في القسم SELECT full_name, salary, NTH_VALUE(salary, 2) OVER ( PARTITION BY dept_id ORDER BY salary DESC ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING ) AS second_highest FROM employee;
↔️ LAG و LEAD — الصفوف المجاورة

LAG and LEAD — Accessing Previous and Next Row Values

الـ LAG بيجيب قيمة من الصف السابق (N صفوف). الـ LEAD من الصف التالي. مفيدان جداً لحساب الفرق بين فترتين زمنيتين.

LAG / LEAD — Time Series Analysis
-- ══ LAG: مقارنة المبيعات بالشهر السابق ══ SELECT month, revenue, LAG(revenue, 1) OVER (ORDER BY month) AS prev_month_revenue, revenue - LAG(revenue, 1) OVER (ORDER BY month) AS month_diff, ROUND( (revenue - LAG(revenue,1) OVER (ORDER BY month)) * 100.0 / NULLIF(LAG(revenue,1) OVER (ORDER BY month), 0), 2 ) AS pct_change FROM monthly_sales ORDER BY month; -- ══ LAG مع Default Value ══ -- LAG(col, offset, default) — لو مفيش صف سابق → يرجع default بدل NULL LAG(revenue, 1, 0) OVER (ORDER BY month) -- 0 بدل NULL للشهر الأول -- ══ LEAD: توقع الطلب القادم لكل عميل ══ SELECT customer_id, order_date, total_amount, LEAD(order_date, 1) OVER ( PARTITION BY customer_id ORDER BY order_date ) AS next_order_date, LEAD(order_date, 1) OVER ( PARTITION BY customer_id ORDER BY order_date ) - order_date AS days_between_orders FROM orders ORDER BY customer_id, order_date; -- ══ LAG لكشف التغييرات المتتالية ══ -- الموظفين اللي اتغيّر قسمهم SELECT emp_id, dept_id, changed_at, LAG(dept_id) OVER (PARTITION BY emp_id ORDER BY changed_at) AS prev_dept FROM employee_dept_history;
📈 Aggregate Window Functions — Running Totals

SUM, AVG, COUNT — Used as Window Functions for Running Calculations

Running Totals and Moving Averages
-- ══ Running Total (مجموع تراكمي) ══ SELECT order_date, total_amount, SUM(total_amount) OVER (ORDER BY order_date) AS running_total, COUNT(*) OVER (ORDER BY order_date) AS running_count FROM orders WHERE status = 'delivered' ORDER BY order_date; -- ══ Running Total بـ PARTITION (مجموع تراكمي لكل فئة) ══ SELECT category_name, order_date, SUM(amount) OVER ( PARTITION BY category_name ORDER BY order_date ) AS category_running_total FROM sales_by_category; -- ══ نسبة تراكمية % ══ SELECT month, revenue, SUM(revenue) OVER (ORDER BY month) AS cumulative, ROUND( SUM(revenue) OVER (ORDER BY month) * 100.0 / SUM(revenue) OVER (), 2 ) AS cumulative_pct FROM monthly_sales;
🖼️ Frame Clause — نطاق النافذة

ROWS and RANGE Frame Clauses — Defining Window Boundaries

الـ Frame بيحدّد أيّ الصفوف تدخل في حساب الـ Window Function بالنسبة للصف الحالي. بيفيد في الـ Moving Average وغيره.

Frame Clause — Moving Average
-- ══ Frame Clause الشكل العام ══ -- { ROWS | RANGE } BETWEEN start AND end -- start/end يمكن أن يكون: -- UNBOUNDED PRECEDING → من أول الـ partition -- N PRECEDING → N صفوف/قيم قبل الحالي -- CURRENT ROW → الصف الحالي -- N FOLLOWING → N صفوف/قيم بعد الحالي -- UNBOUNDED FOLLOWING → حتى آخر الـ partition -- ══ Moving Average (متوسط متحرك 3 أشهر) ══ SELECT month, revenue, ROUND(AVG(revenue) OVER ( ORDER BY month ROWS BETWEEN 2 PRECEDING AND CURRENT ROW ), 2) AS moving_avg_3m, -- الشهر الحالي + 2 سابقين ROUND(AVG(revenue) OVER ( ORDER BY month ROWS BETWEEN 1 PRECEDING AND 1 FOLLOWING ), 2) AS centered_avg -- شهر قبل + الحالي + شهر بعد FROM monthly_sales ORDER BY month; -- ══ ROWS vs RANGE ══ -- ROWS: صفوف فعلية (physical rows) -- RANGE: نطاق بقيمة ORDER BY — كل الصفوف بنفس القيمة = CURRENT ROW -- الـ Default بدون frame clause: RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW

Window Functions — Solutions
-- A: ترتيب الموظفين في قسمهم + فرق الراتب SELECT full_name, dept_id, salary, DENSE_RANK() OVER (PARTITION BY dept_id ORDER BY salary DESC) AS dept_rank, FIRST_VALUE(salary) OVER (PARTITION BY dept_id ORDER BY salary DESC) - salary AS gap_from_top FROM employee ORDER BY dept_id, dept_rank; -- B: أوردرات 2024 + تراكمي + نسبة SELECT order_id, order_date, total_amount, SUM(total_amount) OVER (ORDER BY order_date) AS running_total, ROUND(total_amount * 100.0 / SUM(total_amount) OVER (), 2) AS pct_of_total FROM orders WHERE status = 'delivered' AND order_date BETWEEN '2024-01-01' AND '2024-12-31' ORDER BY order_date; -- C: Moving Average 3 أشهر SELECT month, revenue, ROUND(AVG(revenue) OVER ( ORDER BY month ROWS BETWEEN 2 PRECEDING AND CURRENT ROW ), 2) AS moving_avg_3m FROM ( SELECT DATE_TRUNC('month', order_date) AS month, SUM(total_amount) AS revenue FROM orders WHERE status = 'delivered' GROUP BY DATE_TRUNC('month', order_date) ) AS monthly ORDER BY month;

١. ما الفرق بين RANK() وDENSE_RANK() في حالة التعادل؟
مفيش فرق
RANK() يترك فجوات (1,1,3). DENSE_RANK() بدون فجوات (1,1,2). لـ "أعلى N" استخدم DENSE_RANK.
DENSE_RANK() أسرع
٢. ما الفرق بين OVER() بدون شيء وOVER(PARTITION BY dept_id)؟
OVER() أبطأ
OVER() = النافذة كل الجدول (قيمة واحدة للكل). OVER(PARTITION BY dept_id) = قيمة مستقلة لكل قسم.
مفيش فرق في النتيجة
٣. عايز تعمل "آخر 3 أشهر Moving Average" — أي Frame Clause تستخدم؟
ROWS BETWEEN 3 PRECEDING AND CURRENT ROW
ROWS BETWEEN 2 PRECEDING AND CURRENT ROW (الحالي + 2 سابقين = 3)
RANGE BETWEEN 3 PRECEDING AND CURRENT ROW
